## Введение

**RWSDBv2.1** — это система управления базой данных железнодорожной станции. Приложение предназначено для работы с данными о поездах, пассажирах, билетах, расписании, платформах, сотрудниках и услугах.

### Основные возможности

- Просмотр и редактирование таблиц базы данных
- Фильтрация данных в таблицах
- Добавление, обновление и удаление записей
- Выполнение специальных запросов
- Создание резервных копий базы данных
- Экспорт данных в различные форматы

### Стандарт CUA

Приложение разработано в соответствии со стандартом **CUA (Common User Access)**, который обеспечивает:
- Единообразие управления через клавиатуру
- Предсказуемое поведение интерфейса
- Минимальное время отклика (< 0,1 с)
- Транзакционный характер взаимодействия

## Запуск приложения

### Системные требования

- **ОС:** Windows 10/11, Linux, macOS
- **Браузер:** Chrome 90+, Firefox 88+, Edge 90+
- **Сервер:** PostgreSQL 13+
- **Backend:** Python 3.8+

### Запуск

1. Откройте командную строку (Terminal)
2. Перейдите в директорию проекта:
   ```
   cd C:\VS_Code\RWSDBv2.1
   ```
3. Запустите приложение:
   ```
   npm start
   ```
4. Откройте браузер и перейдите по адресу:
   - **Frontend:** http://localhost:3000
   - **Backend:** http://localhost:8080

## Работа с таблицами

### Выбор таблицы

1. Нажмите **Alt+T** для открытия меню Tables
2. Используйте стрелки **↑/↓** для выбора таблицы
3. Нажмите **Enter** для подтверждения

Или используйте горячие клавиши:
- **Alt+T,S** — Schedule
- **Alt+T,T** — Train
- **Alt+T,P** — Passenger
- и т.д.

### Просмотр данных

**Действие:** Ctrl+V или Operations → View

Данные активной таблицы отображаются в основной области в виде таблицы.

### Добавление записи

**Действие:** Ctrl+A или Operations → Add

1. Открывается диалоговое окно «Add Record»
2. Заполните поля формы
3. Нажмите **Enter** или кнопку **OK** для сохранения
4. Нажмите **Esc** или кнопку **Cancel** для отмены

```
┌─────────────────────────────────────┐
│  Add Record - schedule              │
├─────────────────────────────────────┤
│  Train ID:     [____________]       │
│  Platform ID:  [____________]       │
│  Arrival:      [____________]       │
│  Departure:    [____________]       │
├─────────────────────────────────────┤
│              [Cancel]  [OK]         │
└─────────────────────────────────────┘
```

### Обновление записи

**Действие:** Ctrl+U или Operations → Update

1. Выберите запись в таблице
2. Нажмите **Ctrl+U**
3. Измените необходимые поля
4. Нажмите **Enter** или **OK** для сохранения

### Удаление записи

**Действие:** Ctrl+D или Operations → Delete

1. Выберите запись в таблице
2. Нажмите **Ctrl+D**
3. Введите ID записи для подтверждения
4. Нажмите **Enter** или **Delete** для подтверждения

```
┌─────────────────────────────────────┐
│  ⚠ Delete Record                    │
├─────────────────────────────────────┤
│  Вы уверены, что хотите удалить     │
│  запись ID=5 из таблицы schedule?   │
│                                     │
│  Это действие нельзя отменить.      │
│                                     │
│  Record ID: [____]                  │
├─────────────────────────────────────┤
│           [Cancel]  [Delete]        │
└─────────────────────────────────────┘
```

## Выполнение запросов

### Специальные запросы

**Действие:** Ctrl+Q или Operations → Queries

1. Открывается диалог «Special Queries»
2. Выберите запрос из списка (стрелки ↑/↓)
3. Нажмите **Enter** или **Execute** для выполнения

### Доступные запросы

| ID | Название | Описание |
|----|----------|----------|
| all_trains | All Trains | Все поезда в базе данных |
| all_passengers | All Passengers | Все пассажиры |
| all_tickets | All Tickets | Все билеты |
| all_schedules | All Schedules | Все расписания |
| all_employees | All Employees | Все сотрудники |
| all_platforms | All Platforms | Все платформы |
| all_services | All Services | Все услуги |

### Сохранение результатов запроса

**Действие:** Ctrl+S или Operations → Save Query

1. После выполнения запроса нажмите **Ctrl+S**
2. Подтвердите сохранение
3. Файл будет сохранён в:
   ```
   C:\VS_Code\RWSDBv2.1\data\exports\query_result.xlsx
   ```

## Создание резервных копий

**Действие:** Ctrl+B или Operations → Backup

**Требования:**
- Права суперпользователя
- Установленный pg_dump (PostgreSQL)

### Процесс создания бэкапа

1. Нажмите **Ctrl+B**
2. Введите пароль суперпользователя
3. Нажмите **Enter** или **Create Backup**
4. Дождитесь завершения операции

```
┌─────────────────────────────────────┐
│  📦 Create Database Backup          │
├─────────────────────────────────────┤
│  Creating a backup requires         │
│  superuser privileges.              │
│                                     │
│  ⚠️ Important: Only superuser       │
│     can perform this operation.     │
│                                     │
│  Password: [********] 👁            │
├─────────────────────────────────────┤
│          [Cancel]  [Create Backup]  │
└─────────────────────────────────────┘
```

### Расположение бэкапов

```
C:\VS_Code\RWSDBv2.1\data\backups\backup_YYYYMMDD_HHMMSS.dump
```

## Экспорт данных

### Форматы экспорта

- **Excel (.xlsx)** — таблицы Excel
- **CSV (.csv)** — текстовый формат
- **JSON (.json)** — структурированный формат

### Экспорт таблицы

1. Выберите таблицу
2. Откройте меню Operations
3. Выберите Export (если доступно)
4. Выберите формат
5. Файл будет сохранён в директорию exports

## Приложение А. Требования к интерфейсу

### А.1 Общие требования

| Требование | Описание |
|------------|----------|
| **Здоровье оператора** | Важнее эстетики |
| **Без скроллинга в формах** | Информация в диалогах не скроллируется |
| **Транзакционность** | Без явного подтверждения ничего не происходит |
| **Нет прогрессивных предложений** | Программа не управляет действиями оператора |
| **Время отклика** | Не более 0,1 с |

### А.2 Клавиатурные события (CUA)

| Событие | Действие |
|---------|----------|
| F10 | Активация меню |
| Alt+буква | Открытие меню |
| Ctrl+буква | Выполнение команды |
| Enter | Подтверждение |
| Esc | Отмена |
| Tab | Переход между полями |
| Shift+Tab | Обратный переход |
| Стрелки | Навигация в меню/списках |

### А.3 Диалоговые окна

Все диалоговые окна являются **статическими** и содержат:

1. **Заголовок** с названием операции
2. **Основную область** с полями ввода
3. **Панель действий** с кнопками OK/Cancel

### А.4 Сообщения об ошибках

| Тип | Описание |
|-----|----------|
| **Ошибка** | Красный фон, иконка ⚠️ |
| **Предупреждение** | Жёлтый фон, иконка ⚠️ |
| **Успех** | Зелёный фон, иконка ✓ |
| **Информация** | Синий фон, иконка ℹ️ |
