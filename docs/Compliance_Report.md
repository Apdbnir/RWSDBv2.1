### 1.2 Формат JSON ✅

**Требование:** Тела ответов сервера, так же, как и тела запросов должны быть представлены в формате JSON.

**Реализация:**
- Все запросы с телом (POST, PUT, PATCH) ожидают JSON
- Все ответы сервера возвращают JSON
- Content-Type: `application/json`

**Пример запроса:**
```json

{
  "full_name": "Иванов Иван Иванович",
  "position": "Engineer",
  "salary": 75000
}
```

**Пример ответа:**
```json
{
  "employee_id": 101,
  "full_name": "Иванов Иван Иванович",
  "position": "Engineer",
  "salary": 75000,
  "hire_date": "2026-02-24",
  ...
}
```

### 3.3 Аутентификация суперпользователя ✅

**Требование:** Для выполнения действий от имени суперпользователя приложение должно запрашивать пароль.

**Реализация:**
- Пароль передается в заголовке `Authorization: Bearer {password}`
- Проверка через хеширование (SHA256)
- Пароль хранится в `config.json`

**Код:**
```python
def _authenticate_superuser(self):
    auth_header = self.headers.get('Authorization')
    if not auth_header or not auth_header.startswith('Bearer '):
        return False
    token = auth_header.split(' ')[1]
    return hashlib.sha256(token.encode()).hexdigest() == self.admin_password_hash
```

**Статус:** ✅ **ПОЛНОЕ СООТВЕТСТВИЕ**

```python
    db_table = self.get_table_name(table_name)
```
