#### 2.1.3. Классификация таблиц

#### 2.1.4. Ролевая модель

**Роль: Пользователь**
- Просмотр всех таблиц (GET запросы)
- Фильтрация содержимого таблиц
- Добавление записей в операционные таблицы (POST)
- Обновление записей в операционных таблицах (PUT)
- Удаление записей из операционных таблиц (DELETE)
- Выполнение специальных запросов (SELECT)
- Сохранение результатов в файлы (JSON, CSV, Excel)

**Ограничения для пользователя:**
- Запрещено изменение справочных таблиц
- Запрещено создание бэкапов БД

**Роль: Суперпользователь**
- Все права обычного пользователя
- Редактирование справочных таблиц
- Создание бэкапов базы данных

**Аутентификация суперпользователя:**
- Пароль: 4444
- Передается в заголовке: `Authorization: Bearer {password}`

### 2.2. Программирование серверной части

#### 2.2.1. Скрипты базы данных

**Скрипт создания схемы:**
- Файл: `database/postgres_create_schema_lab1.sql`
- Создает все таблицы, ограничения, индексы, представления

**Скрипт наполнения справочников:**
- Файл: `database/populate_lookup_tables_lab1.sql`
- Наполняет справочные таблицы начальными данными

**Скрипт генерации данных:**
- Файл: `database/generate_main_data_lab1.sql`
- Генерирует тестовые данные:
  - passenger: 300 записей
  - schedule: 500 записей
  - ticket: 500 записей
  - appointment: 100 записей
  - work: 200 записей

#### 2.2.2. Реализация HTTP сервера

Серверная часть реализована на Python с использованием:
- `http.server` - HTTP сервер
- `psycopg2-binary` - библиотека для работы с PostgreSQL (на основе libpq)
- `json` - формат обмена данными

**Структура server.py:**
- Класс `DatabaseManager` - управление подключением к БД
- Класс `RequestHandler` - обработка HTTP запросов
- Функция `run_server()` - запуск сервера

#### 2.2.3. Примеры запросов

**GET - Получение расписания:**
```
GET http://localhost:8080/api/schedule
```

**GET - Фильтрация:**
```
GET http://localhost:8080/api/schedule?train_number=001А
```

**POST - Создание билета:**
```json
POST /api/ticket
Content-Type: application/json

{
  "ticket_number": "TKT000501",
  "carriage_number": "5",
  "ticket_price": 1500.00,
  "seat_number": "12В",
  "passenger_number": "PASS000001",
  "train_number": "001А"
}
```

**PUT - Обновление:**
```json
PUT /api/ticket/TKT000501
Content-Type: application/json

{
  "ticket_price": 1600.00
}
```

**DELETE - Удаление:**
```
DELETE /api/ticket/TKT000501
```

#### 2.2.4. Тестирование сервера

Тестирование выполнено с помощью:
- psql для проверки БД
- curl для HTTP запросов
- Браузера для GET запросов

**Пример тестирования:**
```bash
# Проверка получения данных
curl http://localhost:8080/api/schedule

# Проверка статистики
curl http://localhost:8080/api/statistics

# Проверка создания бэкапа (суперпользователь)
curl -X POST http://localhost:8080/api/backup \
  -H "Authorization: Bearer 4444"
```
