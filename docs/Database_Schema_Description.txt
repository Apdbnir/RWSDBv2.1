================================================================================
ОПИСАНИЕ СХЕМЫ БАЗЫ ДАННЫХ
«Железнодорожный вокзал»
================================================================================

Таблица train (Поезд)
----------------------
Таблица представляет поезда железнодорожного вокзала.

Описание имен таблицы train:
  - train_number: номер поезда. Первичный ключ, уникальный идентификатор;
  - speed: максимальная скорость поезда;
  - year_of_manufacture: год выпуска поезда;
  - type: тип поезда (пассажирский, грузовой, скоростной и т.д.);
  - number_of_cars: количество вагонов в поезде.


Таблица platform (Платформа)
-----------------------------
Таблица представляет железнодорожные платформы вокзала.

Описание имен таблицы platform:
  - platform_number: номер платформы. Первичный ключ, уникальный идентификатор;
  - capacity: вместимость платформы (количество пассажиров);
  - location: расположение платформы (терминал, сектор);
  - number_of_tracks: количество путей на платформе.


Таблица passenger (Пассажир)
-----------------------------
Таблица представляет пассажиров железнодорожного вокзала.

Описание имен таблицы passenger:
  - passenger_number: номер пассажира. Первичный ключ, уникальный идентификатор;
  - passport_number: паспортный номер пассажира;
  - full_name: полное имя пассажира (ФИО);
  - mobile_phone: мобильный телефон пассажира;
  - feature: особенность (льготная категория, особые потребности и т.д.).


Таблица ticket (Билет)
-----------------------
Таблица представляет билеты на поезда.

Описание имен таблицы ticket:
  - ticket_number: номер билета. Первичный ключ, уникальный идентификатор;
  - carriage_number: номер вагона;
  - ticket_price: стоимость билета;
  - seat_number: номер места;
  - passenger_number: номер пассажира. Внешний ключ -> passenger.passenger_number.


Таблица schedule (Расписание)
------------------------------
Таблица представляет расписание движения поездов.

Описание имен таблицы schedule:
  - schedule_number: номер расписания. Первичный ключ, уникальный идентификатор;
  - arrival_time: время прибытия поезда;
  - departure_time: время отправления поезда;
  - carriage_numbering: нумерация вагонов для рейса;
  - date: дата рейса;
  - train_number: номер поезда. Внешний ключ -> train.train_number;
  - platform_number: номер платформы. Внешний ключ -> platform.platform_number;
  - ticket_number: номер билета. Внешний ключ -> ticket.ticket_number.


Таблица service (Услуга)
-------------------------
Таблица представляет услуги, предоставляемые на железнодорожном вокзале.

Описание имен таблицы service:
  - service_number: номер услуги. Первичный ключ, уникальный идентификатор;
  - service_name: название услуги;
  - price: стоимость услуги;
  - type: тип услуги (камера хранения, зал ожидания, питание и т.д.);
  - date: дата предоставления услуги.


Таблица employee (Сотрудник)
-----------------------------
Таблица представляет сотрудников железнодорожного вокзала.

Описание имен таблицы employee:
  - employee_number: номер сотрудника. Первичный ключ, уникальный идентификатор;
  - full_name: полное имя сотрудника (ФИО);
  - position: должность сотрудника;
  - work_experience: стаж работы сотрудника;
  - passport_information: паспортные данные сотрудника.


Таблица appointment (Назначение)
---------------------------------
Таблица представляет назначения сотрудников на услуги.

Описание имен таблицы appointment:
  - employee_number: номер сотрудника. Первичный ключ (составной), 
                     Внешний ключ -> employee.employee_number;
  - service_number: номер услуги. Первичный ключ (составной), 
                    Внешний ключ -> service.service_number.


Таблица work (Работа)
----------------------
Таблица представляет назначения сотрудников на поезда.

Описание имен таблицы work:
  - train_number: номер поезда. Первичный ключ (составной), 
                 Внешний ключ -> train.train_number;
  - employee_number: номер сотрудника. Первичный ключ (составной), 
                     Внешний ключ -> employee.employee_number.

СПРАВОЧНЫЕ ТАБЛИЦЫ (Lookup Tables)
-----------------------------------
В данную категорию выделены следующие таблицы, так как они содержат 
предварительно определенные данные, используемые для поиска и сопоставления 
значений в других таблицах:

  1. train (Поезда)
     - Содержит фиксированный перечень подвижного состава
     - Изменения происходят редко (добавление/списание поездов)
     - Используется для ссылки из таблицы schedule
     - Только суперпользователь может добавлять/удалять поезда

  2. platform (Платформы)
     - Содержит стационарную инфраструктуру вокзала
     - Данные практически неизменны после создания
     - Используется для распределения рейсов по платформам
     - Только суперпользователь может модифицировать

  3. service (Услуги)
     - Содержит утвержденный перечень услуг вокзала
     - Изменения требуют административного решения
     - Используется для назначения услуг сотрудникам
     - Только суперпользователь может управлять списком


ОСНОВНАЯ ТАБЛИЦА
-----------------
Таблица: schedule (Расписание)

В качестве основной таблицы выделена таблица schedule, так как:
  - Содержит основные бизнес-события (рейсы поездов)
  - Данные в ней динамичны и постоянно изменяются (ежедневные рейсы)
  - Интегрирует информацию из справочных таблиц (поезда, платформы, билеты)
  - Является центральным элементом операционной деятельности вокзала


ОПЕРАЦИОННЫЕ ТАБЛИЦЫ
---------------------
  - ticket      - Билеты (динамически пополняемая таблица продаж)
  - passenger   - Пассажиры (данные о клиентах вокзала)
  - employee    - Сотрудники (кадровый состав вокзала)
  - appointment - Назначения на услуги (связь сотрудников с услугами)
  - work        - Назначения на поезда (связь сотрудников с поездами)
