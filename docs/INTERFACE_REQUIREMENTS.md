## Введение

Настоящий документ определяет требования к внешним интерфейсам системы управления базой данных железнодорожной станции RWSDBv2.1. Документ разработан в рамках лабораторной работы №2 и соответствует стандарту CUA (Common User Access).

### Назначение документа

- Определение требований к пользовательскому интерфейсу
- Описание аппаратных интерфейсов
- Спецификация программных интерфейсов
- Определение требований к производительности

### Область применения

Документ предназначен для:
- Разработчиков пользовательского интерфейса
- Тестировщиков
- Специалистов по технической поддержке
- Пользователей системы

## Пользовательские интерфейсы

### 2.1 Общие требования

#### 2.1.1 Принципы проектирования UI

| Принцип | Описание |
|---------|----------|
| **Здоровье оператора** | Важнее эстетики. Минимизация нагрузки на зрение и психику |
| **Без скроллинга в формах** | Информация в диалоговых окнах типа «Форма» не скроллируется |
| **Транзакционность** | Без явного подтверждения ничего не происходит |
| **Отказ от прогрессивных предложений** | Программа не управляет действиями оператора |
| **Время отклика** | Не более 0,1 с на любое событие |

#### 2.1.2 Стиль интерфейса

- **Классический меню-ориентированный интерфейс**
- **Статические диалоговые окна** (без динамической подгрузки контента)
- **Текстовый режим** с минимальным использованием графики
- **Высокая контрастность** для читаемости

## Программные интерфейсы

### 4.1 Клиент-серверный интерфейс

#### 4.1.1 Протокол взаимодействия

- **Протокол:** HTTP/1.1
- **Формат данных:** JSON
- **Кодировка:** UTF-8

#### 4.1.2 Базовый URL API

```
http://localhost:8080/api
```

#### 4.1.3 Конечные точки API

| Метод | Endpoint | Описание |
|-------|----------|----------|
| GET | /api/{table} | Получить данные таблицы |
| GET | /api/{table}/{id} | Получить запись по ID |
| POST | /api/{table} | Создать запись |
| PUT | /api/{table}/{id} | Обновить запись |
| DELETE | /api/{table}/{id} | Удалить запись |
| GET | /api/statistics | Получить статистику |
| POST | /api/backup | Создать резервную копию |
| POST | /api/export | Экспортировать данные |
| POST | /api/queries | Выполнить запрос |

### 4.2 Интерфейс с базой данных

#### 4.2.1 СУБД

- **Тип:** PostgreSQL
- **Версия:** 13 или выше
- **Протокол:** PostgreSQL Wire Protocol

#### 4.2.2 Параметры подключения

```
Host: localhost
Port: 5432
Database: railway_station
User: postgres
Password: [конфигурируется]
```

#### 4.2.3 Таблицы базы данных

| Таблица | Описание | CRUD |
|---------|----------|------|
| schedule | Расписание | Все пользователи |
| train | Поезда | Только чтение / Суперпользователь |
| platform | Платформы | Только чтение / Суперпользователь |
| passenger | Пассажиры | Все пользователи |
| ticket | Билеты | Все пользователи |
| employee | Сотрудники | Только чтение / Суперпользователь |
| service | Услуги | Только чтение / Суперпользователь |

### 4.3 Библиотеки и фреймворки

#### 4.3.1 Клиентская часть (Frontend)

| Библиотека | Версия | Назначение |
